ShieldSync Inc. — Infrastructure Overview
Version 2.5 | Last Updated: October 2024

1. CLOUD ARCHITECTURE

ShieldSync operates a multi-region cloud-native architecture on Amazon Web Services (AWS). Primary production workloads run in us-east-1 (N. Virginia) with a hot standby in eu-west-1 (Ireland) for European customers. All infrastructure is defined as code using Terraform and deployed through automated CI/CD pipelines (GitHub Actions).

2. NETWORK SECURITY

Our network architecture implements defense-in-depth principles:
- Web Application Firewall (WAF): AWS WAF with custom rule sets blocking OWASP Top 10 attack patterns, rate limiting, and geo-blocking.
- DDoS Protection: Cloudflare Enterprise with automatic mitigation up to 100 Tbps.
- Network Segmentation: VPC with separate subnets for web servers, application servers, and databases. No direct internet access to application or database tiers.
- Intrusion Detection: CrowdStrike Falcon deployed on all server instances with real-time threat detection and automated response.
- Firewall Rules: Default-deny policy on all security groups. Ingress is limited to HTTPS (443) on load balancers. SSH access is through a bastion host with MFA.
- TLS Configuration: TLS 1.3 enforced on all endpoints. HSTS headers with max-age of 31536000 seconds. Certificate transparency monitoring enabled.

3. DISASTER RECOVERY

- Recovery Time Objective (RTO): 4 hours for full platform recovery.
- Recovery Point Objective (RPO): 1 hour maximum data loss.
- Database: PostgreSQL with streaming replication to standby region. Automated failover via AWS RDS Multi-AZ.
- Object Storage: S3 cross-region replication enabled for all customer data buckets.
- Testing: Full disaster recovery drills are conducted quarterly. Last drill completed successfully on September 15, 2024 with actual RTO of 2.5 hours and RPO of 23 minutes.

4. MONITORING AND LOGGING

All system logs are centralized in Splunk with a 12-month retention period. Application performance is monitored via Datadog with alerting thresholds on latency (p99 < 500ms), error rates (< 0.1%), and availability (> 99.95%). Security events are correlated through our SIEM with automated playbooks for common threat patterns. Uptime over the past 12 months: 99.97%.
